# UiPath Claude Code - Quick Start Guide

## Install

```bash
git clone <repo-url>
cd uipath-builder-agent
python -m venv .venv
# Windows:
.\.venv\Scripts\Activate.ps1
# Linux/Mac:
source .venv/bin/activate
pip install -e ".[dev]"
git submodule update --init --recursive
```

## UiPath CLI Setup (Optional - for workflow validation)

The `uip` CLI is required for `uip rpa get-errors --use-studio` validation.

```bash
# Install via npm
npm install -g @uipath/cli

# Install required tools and dependencies
npm install -g @uipath/rpa-tool @uipath/common @uipath/solutionpackager-tool-core commander
```

**Requirements:**
- Node.js 18+
- UiPath Studio Desktop **26.2+** (required for `--use-studio` commands)
- Studio must be running for IPC commands

## AWS Setup (Bedrock)

```bash
aws sts get-caller-identity
```

If this fails, configure credentials first.

Optional overrides:

```bash
# Defaults
set AWS_REGION=us-east-1
set UIPATH_CLAUDE_MODEL=anthropic.claude-3-sonnet-20240229-v1:0
```

## Start Chat

```bash
uipath-claude chat
```

Exit with `exit` or `quit`.

Streaming is enabled by default in chat. Use:

```bash
uipath-claude chat --no-stream
```

Or set:

```bash
# 0/false/no disables streaming
set UIPATH_CHAT_STREAM=0
```

Optional response-output mode:

```bash
# auto (default): after the run, avoid dumping huge assistant bodies when the reply
#     contains file blocks (streaming deltas are unchanged)
# full: always print full assistant text after the run
# quiet: suppress extra assistant body re-print after streaming (deltas still stream if UIPATH_CHAT_STREAM=1)
set UIPATH_CHAT_OUTPUT_MODE=auto
```

## Chat file output (on-disk)

When the model replies with file blocks in one of these formats, the CLI **writes files** under `generated/chat/<chat-id>/` (or `UIPATH_CHAT_OUTPUT_DIR/<chat-id>/`):

```
<<<UIPATH_FILE path="demo/Main.xaml">>>
...XAML body...
<<<END_UIPATH_FILE>>>
```

Or a markdown fence whose **first line** is `path: <relative path>`:

````
```xml
path: demo/Main.xaml
...file body...
```
````

- `UIPATH_CHAT_OUTPUT_DIR` — optional root directory for writes (default: `./generated/chat`).
- `UIPATH_CHAT_SESSION_ID` — optional explicit trace ID folder name (default: autogenerated timestamp+UUID).
- `UIPATH_CHAT_MATERIALIZE=0` — disable writing from chat replies.
- `UIPATH_CHAT_DEBUG_SKILLS=1` — print top skill matches/scores for each prompt.

For full BA/SA/Dev/QA artifacts (PDD/SDD/QA + scaffold), use `/bootstrap "<request>"` or `uipath-claude start-project "Name"`; those paths are separate from chat blocks.

## Slash Commands

- `/help` - Show available commands
- `/status` - Session status
- `/skills` - Skills summary
- `/analyze [path]` - Analyze project path
- `/bootstrap "<request>"` - Run BA→SA→Dev→QA and write docs + generated scaffold
- `/repair-restore` - Show deterministic steps for NuGet restore lock/permission errors
- `/chat` - Indicates you are already in chat mode
